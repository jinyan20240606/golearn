# jilei

1. Linux shell 文件描述符：0、1、2、&1 完整说明
   1. 三个标准流固定编号
      1. 0 = stdin 标准输入：键盘输入，程序读取数据用。
      2. 1 = stdout 标准输出：程序正常打印、fmt.Println、业务日志，默认打印到终端。
      3. 2 = stderr 标准错误：panic、崩溃堆栈、错误打印、命令报错，默认也打印到终端，但和 stdout 是两条独立数据流。
   2. > 重定向：> 文件：默认把1 号流（stdout） 写入文件
   3. &1：不是数字 1，是引用 1 号流的目标位置
      1. 如2>&1 翻译：把 标准错误 (2) 重定向到 和标准输出 (1) 同一个输出位置。
2. `GEO_ENV=dev /Users/jinyan1/Desktop/work/backend/geo/bin/geo api > /tmp/geo.log 2>&1 & sleep 2 && curl -s http://localhost:8080/health`
   1. > /tmp/geo.log: 标准输出 stdout（fmt 打印、正常业务日志）重定向写入文件 /tmp/geo.log，不再打印控制台。
   2. 2>&1
      1. 标准错误 stderr（崩溃堆栈、panic、error 输出）重定向到标准输出，最终全部日志统一写入同一个 log 文件。
      2. 不加这条，报错会继续弹控制台，不会落文件。
   3. 末尾 &: 把进程丢到后台异步运行，终端不阻塞，可以继续输入下一条命令。
3. `pkill -f "geo api" 2>/dev/null; echo "stopped"`
   1. pkill：按进程匹配名杀死进程
   2. -f：匹配完整命令行，不只是程序名,不加 -f 只能搜进程名 geo
   3. 2>/dev/null
      1. 2 = 标准错误流 stderr
      2. /dev/null 系统黑洞，写入的数据直接丢弃
4. 查看进程
   1. ps aux | grep 进程名：列出所有进程，过滤目标，能看到完整启动命令
   2. ps -p PID：显示该进程的详细信息 
   3. pgrep 程序名：只输出 PID，匹配二进制短名
   4. pgrep -f "关键词"：匹配完整启动命令（区分 geo api /geo task）
   5. pidof geo：直接拿二进制名查 PID
   6. cat /proc/$pid/comm：查看内核短进程名
   7. cat /proc/$pid/cmdline：查看完整启动参数
5. 杀掉进程
   1. pkill 程序名：杀所有同二进制进程（geo api、geo task 一起杀）
   2. pkill -f "geo api"：精准匹配完整命令，只杀 api 服务
   3. pkill -f "xxx" 2>/dev/null：找不到进程不抛报错
   4. kill -15 PID：优雅关闭（优先用）
   5. kill -9 PID：强制暴力杀，不推荐
