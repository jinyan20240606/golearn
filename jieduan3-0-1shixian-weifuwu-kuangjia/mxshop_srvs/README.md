# mxshop_srvs 微服务项目分析

本项目是一个基于 gRPC 的电商微服务后端项目，包含了多个独立的微服务模块。

## 微服务模块划分

1. **user_srv (用户服务)**
   - 负责用户的注册、登录、信息查询等核心用户功能。
2. **goods_srv (商品服务)**
   - 负责商品信息的管理，包括商品分类、品牌、商品详情等。
   - 包含与 Elasticsearch (ES) 的集成，用于商品搜索。
3. **inventory_srv (库存服务)**
   - 负责商品库存的扣减、归还、查询等。
   - 包含分布式锁（Redis Lock）的实现，保证库存操作的原子性。
4. **order_srv (订单服务)**
   - 负责订单的创建、查询、状态流转等。
   - 集成了 RocketMQ 用于处理分布式事务和延迟消息（如超时未支付取消订单）。
   - 包含 OpenTracing (otgrpc) 链路追踪的集成。
5. **userop_srv (用户操作服务)**
   - 负责用户的附加操作，如收货地址管理、用户留言、商品收藏等。

## 通用子目录结构分析

每个微服务模块内部采用了类似的分层架构设计，主要包含以下子目录：

- **config/**: 定义配置文件的结构体（如 MySQL、Redis、Consul 等配置项）。
- **global/**: 定义全局变量，如全局的 DB 实例、Redis 客户端、配置对象等。
- **handler/**: gRPC 接口的具体实现层，处理具体的业务逻辑。
- **initialize/**: 服务的初始化逻辑，包括读取配置、初始化日志、连接数据库、连接 ES 等。
- **model/**: 数据库模型层，定义 GORM 的实体结构体及表结构。
- **proto/**: 存放 Protocol Buffers (`.proto`) 文件以及生成的 `.pb.go` 代码，定义了微服务的 RPC 接口。
- **tests/**: 单元测试和接口测试代码。
- **utils/**: 通用工具包，如动态获取可用端口、Consul 服务注册与发现、Redis 分布式锁等。
- **main.go**: 微服务的启动入口，负责启动 gRPC 服务并注册到 Consul。
- **config-debug.yaml / config-pro.yaml**: 开发环境和生产环境的配置文件。
