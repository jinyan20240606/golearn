## 接口

5. Go 处处是接口 = 万物皆可按 “行为” 归类
   1. 接口就是一堆方法的集合体，go中认为属性不重要，只有行为才重要，只要满足了我的一堆方法，就符合我的接口类型
   2. 任何一个结构体把这组方法实现后，都可以实例化给这个接口类型，而且不需要显示的指明动态语言php抽象类那样class Duck implements Animal {} 显示指明实现的哪个接口
   3. 接口的定义：interface Animal {}
6. 空接口：空接口 = 什么方法都不需要
   1.  interface {} = 万能垃圾桶 / 万能类型 / 任何东西都能装 ----> 相当于ts中的any类型，any自定义类型就是在go中这样封装的
7. 鸭子类型：像鸭子一样走路、叫，它就是鸭子，翻译成 Go：只要一个类型拥有接口里的所有方法，它就自动实现了这个接口
   1. 鸭子类型强调的是事物的外部的行为即暴露的方法，而不是内部的结构属性
8. 为什么说「所有类型都实现了空接口」？
   1. 因为空接口的要求是，什么方法都不用实现！那所有类型天然就满足空接口。
   2. 如下面
      ```go
      var a interface{} // 空接口
      a = 10            // 装 int ✅
      a = "hello"       // 装 string ✅

      ```
9.  空接口最常用在哪？
   1. 函数接收任意类型参数 `func printAny(v interface{}) { fmt.Println(v) }`
   2. 存储任意类型的数据: `m := map[string]interface{}{} m["name"] = "张三" m["age"] = 18`
   3. 不知道传什么类型的时候用
10. 接口的零值 = nil
    1.  接口底层是两个指针：类型指针 + 数据指针，没赋值时，接口就是 nil
        1.  var a Animal；fmt.Println(a == nil) // true


### 终极总结（超级好记）

1. 鸭子类型
方法对，就是同一类 → 不用声明实现，自动满足
2. 空接口 interface {}
没有任何方法要求 → 所有类型都能放 → 万能类型
3. Go 为什么说处处是接口？
因为接口定义行为，而 Go 设计哲学就是：用组合、用行为，来组织代码不是靠继承，而是靠鸭子类型。
4. **精炼一句话**：空接口 interface {} 是 Go 的万能类型，任何值都能放；Go 的接口是鸭子类型，只要方法齐全，就属于这个接口类型


### 断言

- 断言方法很重要，要真正理解断言

1. Go 为什么要搞出「类型断言」？（设计原因）
   1. Go 接口是 隐式实现，多态很强，但有个短板
   2. Go 接口特点：
      1. 不用写 implements
      2. 只要方法匹配，自动实现接口
      3. 一个接口可以装无数种不同结构体
   3. 问题来了：
      1. 接口只给你看通用方法，但我想用具体结构体独有字段怎么办？
      2. 于是必须有类型断言：向下转型，拿到具体类型。
2. 多态场景下，需要区分具体子类
   1. 举例：一个动物接口 Animal可以装 Dog、Cat、Bird
   2. 统一按 Run() 多态调用没问题；但我想 Dog 独有方法 Wang ()、Cat 独有 Miao ()👉 必须类型断言判断是哪个结构体，再调用独有功能。
3. 空接口 interface {} 万能容器，必须靠断言还原
   1. Go 里 interface{} 能装任意类型：存进去是万能的，取出来不知道是什么类型，只能靠类型断言还原成原本类型
   2. 比如：JSON 解析、参数泛型、容器存储，全靠断言。
4. Go 没有传统面向对象的「继承、向下转型」，
   1. Go 不用继承，只用接口 + 组合；类型断言 就是 Go 版的「向下类型转换」。


- **语法细节**
  - `一条死规则：类型断言，只能用在「接口类型变量」上非接口变量，不能做类型断言，语法直接报错。`
  - 两种标准语法格式: 
    - 语法写法：`v, ok := i.(T)`
      - T：你猜测的具体类型（结构体指针、int、string、自定义类型都行）
      - v：断言成功后，拿到的具体类型对应的原真实数据
      - ok：布尔值，是否断言成功
    - 格式 1：单返回值（失败直接 panic）
      - 真实值 := 接口变量.(具体类型)
      - 断言成功：返回底层存的真实类型值,断言失败：程序直接崩溃 panic
    - 格式 2：双返回值（推荐，安全不崩溃）
      - 真实值, ok := 接口变量.(具体类型)
      - ok == true：断言成功，真实值 可用
      - ok == false：断言失败，真实值 是类型零值，程序不崩溃
  - **大致原理**：接口 = 类型 + 数据，类型断言 = 取出接口里的真实类型对应的数据
- 示例
```go
// 第一步:定义一个接口
type MyAddr interface {
	GetPort() int
}
// 第二步：写一个结构体实现这个接口
// 真实的结构体
type TCPAddr struct {
	IP   string
	Port int
}

// 实现接口方法
func (t *TCPAddr) GetPort() int {
	return t.Port
}

// 第三步：把结构体放进接口里
// 接口变量
var addr MyAddr

// 把结构体赋值给接口（Go 自动装箱）
addr = &TCPAddr{IP: "127.0.0.1", Port: 8080}
//现在 addr 接口里存的是：
//类型：*TCPAddr
//数据：&TCPAddr{IP:"127.0.0.1", Port:8080}

// 最终使用类型断言！
tcpAddr, ok := addr.(*TCPAddr)

if ok {// 如果不使用类型断言，addr自己是永远取不到Port的
	fmt.Println("端口：", tcpAddr.Port)
}
```
