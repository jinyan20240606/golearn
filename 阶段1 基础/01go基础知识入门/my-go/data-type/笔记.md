# 数据类型笔记
## 一、先明确：Go 数值类型的核心特点
强类型 + 静态类型：一旦声明变量类型（如 int、float64），无法隐式转换为其他数值类型（比如 int 不能直接加 float64）；
分精确宽度 / 不精确宽度：整数类型分 “固定位数”（如 int8、uint32）和 “适配平台位数”（如 int、uint）；
无隐式类型提升：比如 int8 + int16 会编译报错，必须显式转换为同一类型。




## 二、整数类型（最常用）
整数类型按「符号」「位数」划分，核心用于表示整数（正数、负数、0）。
1. 有符号整数（支持正 / 负 / 0）
类型	位数	取值范围	适用场景
int8	8	-128 ~ 127	小范围整数（如状态码、枚举）
int16	16	-32768 ~ 32767	中等范围整数（如小计数）
int32	32	-2147483648 ~ 2147483647	常用（如整数 ID、计数器）
int64	64	-9223372036854775808 ~ 9223372036854775807	大范围整数（如时间戳、大数计算）
int	随平台	32 位系统 = 32 位，64 位系统 = 64 位	通用整数（无明确位数要求时首选）
2. 无符号整数（仅支持正 / 0）
类型	位数	取值范围	适用场景
uint8	8	0 ~ 255	字节（byte）、ASCII 码（byte 是 uint8 的别名）
uint16	16	0 ~ 65535	无符号中等范围整数
uint32	32	0 ~ 4294967295	无符号常用整数
uint64	64	0 ~ 18446744073709551615	无符号大范围整数
uint	随平台	同 int 位数	通用无符号整数
uintptr	随平台	足够存储指针的整数	底层编程（如指针转换

选型建议
无明确位数要求 → 用 int/uint（适配平台，代码通用）；
明确小范围整数 → 用 int8/uint8（节省内存）；
时间戳、大数计数 → 用 int64（避免溢出）；
字节 / ASCII 处理 → 用 byte（即 uint8）。

### 疑问

   1. 无符号整数 uint8 取值范围是 0 到 255”，核心原因是：uint8 是 8 位无符号整数，8 个二进制位全部用于表示数值，没有符号位，因此取值范围由 2^8 个组合决定

## 三、浮点数类型（常用）

浮点类型用于表示小数，Go 仅支持两种浮点类型（无 float32/float64 之外的类型），遵循 IEEE 754 标准。
类型	位数	精度	       取值范围(这两个数是 float32/64 能表示的最小非零正数和最大正数（负数只是加个符号位，数值部分一致）。)	                  适用场景
float32	32	约 6-7 位有效数字	±1.4e-45 ~ ±3.4e38	精度要求低、节省内存（如简单计算）
float64	64	约 15-17 位有效数字	±4.9e-324 ~ ±1.8e308	通用首选（如科学计算、金额、坐标）


- 1.4e-45 = 1.4 × 10^-45（即 0.000...00014，小数点后有 44 个 0）；
- 3.4e38 = 3.4 × 10^38（即 34 后面跟 37 个 0）；
```go
package main

import "fmt"

func main() {
    var f1 float32 = 3.1415926 // 精度有限，实际存储为3.1415925
    var f2 float64 = 3.141592653589793 // 高精度
    var f3 float64 = 1.23e5 // 科学计数法，等价于123000.0

    fmt.Printf("float32: %.7f\n", f1) // 输出：3.1415925（精度丢失）
    fmt.Printf("float64: %.15f\n", f2) // 输出：3.141592653589793（高精度）
    fmt.Println(f3) // 输出：123000
}
// 
// 注意事项
// float64 是浮点型首选（精度更高，大部分场景无性能损失）；
// 浮点型有精度丢失问题，金额计算禁止用 float（需用 decimal 第三方库或 int64 以分为单位）；
// 不能直接用 == 比较两个浮点数是否相等（需判断差值是否小于极小值，如 1e-9）。
```
## 四、复数类型
```go
// 复数类型用于表示数学中的复数（实部 + 虚部），日常开发几乎不用，仅用于科学计算、信号处理等场景。
// 类型	组成	示例
// complex64	实部 + 虚部都是 float32	complex(1.0, 2.0)
// complex128	实部 + 虚部都是 float64	1 + 2i（字面量）
package main

import "fmt"

func main() {
    // 声明复数
    var c1 complex64 = complex(1, 2) // 1+2i
    var c2 complex128 = 3 + 4i       // 字面量写法（推荐）

    // 获取实部/虚部
    fmt.Println(real(c2)) // 输出：3（实部）
    fmt.Println(imag(c2)) // 输出：4（虚部）
}
```


## 总结

整数型：分有符号 / 无符号、固定位数 / 平台位数，int/int64/byte 是日常最常用的；
浮点型：仅 float32/float64，优先用 float64，金额计算避免用浮点型；
复数型：complex64/complex128，仅科学计算场景用；
核心原则：强类型要求显式转换，按场景选对应精度的类型（够用且不浪费）。
简单记：日常开发优先用 int（整数）、float64（小数）、byte（字节），其他类型按需选用。
